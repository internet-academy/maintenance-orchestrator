# SUCCESS_CRITERIA: Error-Specific Tagging System

## 1. Categorization Clarity (Functional)
- [ ] **Technical Nature**: Tags must distinguish between "Logic" (code behavior) and "Visual" (UI/UX) issues to provide immediate technical context.
- [ ] **Impact Assessment**: Include tags for "Blocking" vs "Cosmetic" to help the team prioritize without looking at a Gantt chart.

## 2. Agent Alignment (Technical)
- [ ] **Tag-to-Agent Mapping**: Tags should align with our specialized agents (`pixel`, `logic`, `ops`, `scout`) to allow for future automated auditing.

## 3. Minimal Friction (Style)
- [ ] **Zero Overlap**: Each tag must have a distinct definition so there is no confusion about which one to apply.
- [ ] **Quick-Scan**: Use a prefix system (e.g., `Type:`, `Impact:`) for high readability on the Kanban board.
